`MeroKitUI` is an optional SwiftUI layer that ships alongside the core SDK — an
`@MainActor` observable `MeroClient` plus ready-made views. It's the native
analog of mero-react's `MeroProvider`/`useMero` + `LoginModal`: wire it in and
you get login → home routing on auth state for free.

```swift
import MeroKit      // core SDK
import MeroKitUI    // SwiftUI frontend
```

(Add the **MeroKitUI** library product alongside **MeroKit** — see
[install](/get-started/quickstart/#install).)

## Drop-in app

`MeroRootView` routes between `LoginView` and `HomeView` based on
`client.isAuthenticated`. Inject a `MeroClient` via `.environmentObject`:

```swift
import MeroKitUI
import SwiftUI

@main
struct MyApp: App {
  @StateObject private var client = MeroClient()

  var body: some Scene {
    WindowGroup {
      MeroRootView()               // routes Login ⇄ Home on auth state
        .environmentObject(client)
    }
  }
}
```

## `MeroClient`

An `@MainActor public final class MeroClient: ObservableObject` wrapping a `Mero`.
Bind its published state directly in your own views.

**Published state** (read-only from views):

```swift
@Published public private(set) var isAuthenticated: Bool
@Published public private(set) var isLoading: Bool
@Published public private(set) var nodeURL: String
@Published public private(set) var username: String
@Published public private(set) var errorMessage: String?
@Published public private(set) var lastRpcResult: String?
```

**Methods** (errors surface via `errorMessage`, so these don't `throw`):

```swift
public init(session: URLSession? = nil,
            tokenStore: @escaping @Sendable () -> any TokenStore = { MemoryTokenStore() })

func login(nodeURL: String, username: String, password: String,
           bootstrapSecret: String? = nil) async
func runSampleRpc(contextId: String, method: String) async
func logout() async
```

Use it in your own SwiftUI view without the bundled screens:

```swift
struct SignInView: View {
  @EnvironmentObject private var client: MeroClient
  @State private var user = ""; @State private var pass = ""

  var body: some View {
    VStack {
      TextField("Username", text: $user)
      SecureField("Password", text: $pass)
      Button("Sign in") {
        Task { await client.login(nodeURL: "https://your-node.example",
                                   username: user, password: pass) }
      }
      if let error = client.errorMessage { Text(error).foregroundStyle(.red) }
    }
    .disabled(client.isLoading)
  }
}
```

## Bundled views

All three are `public` `View`s that read the `MeroClient` from
`@EnvironmentObject`:

| View | Purpose |
| --- | --- |
| `MeroRootView(defaultNodeURL:)` | routes `LoginView` ⇄ `HomeView` on auth state |
| `LoginView(defaultNodeURL:)` | credential form → `client.login(...)` |
| `HomeView(demoContextId:demoMethod:)` | signed-in screen: session info, demo RPC, logout |

<Aside type="note">
  For custom transports (tests, mock backends), construct `MeroClient(session:)`
  with your own `URLSession`. A full SwiftUI sample — plus XCUITest UI tests
  against an in-app mock backend — lives in `Examples/MeroSampleApp`.
</Aside>
